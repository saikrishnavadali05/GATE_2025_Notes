Named Entity Recognition (NER) in NLP

Named Entity Recognition (NER) is the process of identifying and classifying key entities in text into predefined categories such as names of persons, organizations, locations, dates, and more.

Why is NER Important?

  • Extracts structured information from unstructured text
  • Used in information retrieval, question answering, and summarization
  • Helps in building knowledge graphs and semantic search

NER with NLTK

NLTK provides a pre-trained named entity chunker using the ne_chunk function, which recognizes common entities.

10 Examples of Named Entity Recognition

import nltk
from nltk import word_tokenize, pos_tag, ne_chunk

nltk.download('maxent_ne_chunker')
nltk.download('words')
nltk.download('punkt')
nltk.download('averaged_perceptron_tagger')

text = "Apple is looking at buying U.K. startup for $1 billion"

# Example 1: Tokenize and POS tag
tokens = word_tokenize(text)
pos_tags = pos_tag(tokens)

# Example 2: Named entity chunking
named_entities = ne_chunk(pos_tags)
print(named_entities)

# Example 3: Extract named entities as labeled subtrees
for chunk in named_entities:
    if hasattr(chunk, 'label'):
        print(chunk.label(), ' '.join(c[0] for c in chunk))
