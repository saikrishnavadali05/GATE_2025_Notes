Python Header (Shebang & Comments)

How to Use Shebang Lines and Header Comments in Python Scripts

What is a Shebang?

The shebang (#!) is a special character sequence at the very top of a script file that tells Unix-like operating systems which interpreter to use to run the script.

Why Use a Shebang?

  • Allows you to run the script as an executable without explicitly invoking python.
  • Specifies the exact interpreter to use, useful when multiple Python versions are installed.
  • Improves portability across systems.

Common Shebang Lines

#!/usr/bin/env python3
# Preferred for portability; finds python3 in your PATH

#!/usr/bin/python3
# Hardcoded path to python3 interpreter

Example Script Using Shebang

#!/usr/bin/env python3

def greet():
    print("Hello, World!")

if __name__ == "__main__":
    greet()

Making the Script Executable

After adding the shebang, make your script executable with the command:

chmod +x your_script.py

Then run it directly:

./your_script.py

Header Comments in Python Scripts

Besides the shebang, Python scripts often start with a header comment block. This is a multiline comment at the top of the file used to:

  • Provide a decorative or identifying banner
  • Describe the purpose of the script
  • Give usage instructions
  • Optionally include author, date, or version information

Example Header Comment Block

#(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-    % PYTHON %    (**/*+-(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-
#
#
#
# Python Training - Chapter - 3 - Exercise 4 - Question -1 
#
# Description:
#  The code takes two integers and does arithmetic operations on the integers.
# 
#
# Usage:
#  python addition.py 
#
#
#(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-    % PYTHON %    (**/*+-(**/*+-(**/*+-(**/*+-(**/*+-(**/*+-

Why Use Header Comments?

  • Helps readers quickly understand the script's purpose
  • Acts as embedded documentation inside the code
  • Improves maintainability and readability

Important Notes

  • The shebang is required for Unix-like OS executables but ignored on Windows
  • Header comments are for humans, not interpreted by Python
  • Both shebang and header comments usually appear at the very top of the script
