3. Scheduling in 5G NR

Scheduling in 5G NR is the process by which the base station (gNB) allocates radio resources (time and frequency) to User Equipment (UE) for uplink (UL) and downlink (DL) data transmissions.

The gNB is fully in control of scheduling in both UL and DL. The UE waits for instructions from the gNB.

  • Dynamic Scheduling: Resources are allocated on a per-slot or per-TTI basis. Offers flexibility and efficiency.
  • Semi-Persistent Scheduling (SPS): Pre-allocated periodic resources, mainly used for VoIP or low-latency traffic.
  • Configured Grant: Mainly for uplink transmissions with predictable traffic (e.g., IoT, URLLC).
  1. gNB sends a DCI (Downlink Control Information) message via PDCCH.
  2. UE reads the DCI and extracts resource and modulation info.
  3. UE receives data via PDSCH.
  4. UE sends HARQ acknowledgment via PUCCH or PUSCH.
  1. UE sends a SRS (Sounding Reference Signal) or BSR (Buffer Status Report).
  2. gNB allocates uplink resources via DCI.
  3. UE transmits data on PUSCH.
  4. gNB sends HARQ ACK/NACK.

In grant-free or configured grant uplink scheduling, the UE can transmit without waiting for a scheduling grant. This reduces latency and is useful for URLLC scenarios.

5G NR supports flexible slot structures—each slot can carry DL, UL, or special symbols. This enables faster responses and adaptability in scheduling decisions.

  • Beam management with directional antennas
  • Latency-sensitive URLLC scheduling
  • Efficient multiplexing of diverse traffic types (eMBB, mMTC, URLLC)
  • QoS-aware scheduling and prioritization
