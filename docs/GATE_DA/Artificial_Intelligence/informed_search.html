Informed Search

Back to Artificial Intelligence Topics

Informed search algorithms, also known as heuristic search algorithms, utilize additional information about the problem domain to find solutions more efficiently than uninformed search methods. These algorithms use heuristics to estimate the cost of reaching the goal from a given state, guiding the search process.

Overview

Informed search algorithms aim to find the shortest path to the goal by using heuristic functions that provide estimates of the remaining cost to reach the goal. This additional information helps prioritize which paths to explore, improving search efficiency and effectiveness.

Common Informed Search Algorithms

  • Greedy Best-First Search: Prioritizes nodes based on a heuristic function that estimates the cost from the current node to the goal.
  • A* Search: Combines the actual cost from the start node to the current node with the estimated cost from the current node to the goal, using the formula \( f(n) = g(n) + h(n) \).
  • Weighted A*: A variation of A* that uses a weighted heuristic to balance exploration and exploitation.
  • Iterative Deepening A* (IDA*): Combines the concepts of iterative deepening search and A* search to reduce memory usage.
  • Beam Search: Explores a fixed number of best nodes at each level, reducing the search space size.

Heuristic Functions

A heuristic function \( h(n) \) estimates the cost of the cheapest path from node \( n \) to the goal. A good heuristic significantly improves the efficiency of informed search algorithms. Common heuristic functions include:

  • Manhattan Distance: Used in grid-based problems, calculates the sum of the absolute differences of the coordinates.
  • Euclidean Distance: Calculates the straight-line distance between two points in a continuous space.
  • Misplaced Tiles: Used in puzzle problems, counts the number of misplaced tiles compared to the goal state.
  • Hamming Distance: Counts the number of positions at which the corresponding elements are different.

Example: A* Search

The A* search algorithm combines the cost to reach the current node (\( g(n) \)) with the estimated cost to reach the goal (\( h(n) \)) to prioritize nodes in the search space. The formula used is:

\[ f(n) = g(n) + h(n) \]

The algorithm follows these steps:

  1. Initialize the open list with the start node and the closed list as empty.
  2. While the open list is not empty:
    • Select the node with the lowest \( f(n) \) value from the open list.
    • If the selected node is the goal, return the path to the goal.
    • Move the selected node to the closed list.
    • Expand the selected node, and for each child node:
      • If the child node is in the closed list, skip it.
      • If the child node is not in the open list, add it with the calculated \( f(n) \) value.
      • If the child node is already in the open list, update its \( f(n) \) value if the new path is cheaper.
  3. If the open list is empty, return failure (no solution found).

Applications

  • Pathfinding: Used in navigation systems, robotics, and video games to find the shortest path between points.
  • Puzzle Solving: Applied to solve puzzles like the 8-puzzle, 15-puzzle, and Rubik's Cube.
  • Artificial Intelligence: Used in AI for decision making, game playing, and problem-solving.
  • Operations Research: Applied in logistics, scheduling, and optimization problems.

Conclusion

Informed search algorithms leverage heuristic functions to efficiently navigate large search spaces and find optimal solutions. By incorporating domain-specific knowledge, these algorithms outperform uninformed search methods, making them essential tools in artificial intelligence and optimization.
