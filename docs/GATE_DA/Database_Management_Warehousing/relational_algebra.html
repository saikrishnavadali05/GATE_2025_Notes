Relational Algebra

Back to Database Management and Warehousing Topics

Relational algebra is a procedural query language that operates on the relations (tables) of a relational database. It provides a set of operations to manipulate and retrieve data from these relations.

Basic Operations

The fundamental operations of relational algebra include:

  • Select (σ): Selects rows from a relation that satisfy a given predicate.
  • Project (π): Selects specific columns from a relation.
  • Union (⋃): Combines the tuples of two relations and removes duplicates.
  • Set Difference (-): Returns the tuples that are in one relation but not in another.
  • Cartesian Product (×): Combines tuples from two relations in a combinatorial fashion.
  • Rename (ρ): Renames the attributes of a relation.

Select Operation (σ)

The select operation is used to filter rows based on a specified condition. It is denoted by the symbol σ (sigma).

Example:

σcondition(Relation)

If we have a relation Student and we want to select students with age greater than 20, we write:

σage > 20(Student)

Project Operation (π)

The project operation is used to select specific columns from a relation. It is denoted by the symbol π (pi).

Example:

πcolumn1, column2, ...(Relation)

If we want to select only the names and ages of students, we write:

πName, Age(Student)

Union Operation (⋃)

The union operation combines the tuples of two relations and removes duplicates. It is denoted by the symbol ⋃.

Example:

Relation1 ⋃ Relation2

If we have two relations, GraduateStudents and UndergraduateStudents, we can combine them using:

GraduateStudents ⋃ UndergraduateStudents

Set Difference Operation (-)

The set difference operation returns the tuples that are in one relation but not in another. It is denoted by the symbol -.

Example:

Relation1 - Relation2

If we want to find students who are not graduates, we write:

Student - GraduateStudents

Cartesian Product Operation (×)

The Cartesian product operation combines tuples from two relations in a combinatorial fashion. It is denoted by the symbol ×.

Example:

Relation1 × Relation2

If we want to combine Students and Courses relations, we write:

Student × Course

Rename Operation (ρ)

The rename operation is used to rename the attributes of a relation. It is denoted by the symbol ρ (rho).

Example:

ρnew_name(Relation)

If we want to rename the Student relation to Scholar, we write:

ρScholar(Student)

Additional Operations

Relational algebra also includes additional operations such as:

  • Intersection (∩): Returns the tuples that are common to both relations.
  • Join (⨝): Combines related tuples from two relations based on a common attribute.
  • Division (÷): Divides one relation by another.

Conclusion

Relational algebra provides a powerful and expressive framework for querying relational databases. Understanding these operations is crucial for working with relational databases and designing efficient queries.
