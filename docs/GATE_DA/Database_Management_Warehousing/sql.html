SQL

Back to Database Management and Warehousing Topics

SQL (Structured Query Language) is a standard programming language used for managing and manipulating relational databases. SQL is widely used for querying, updating, and managing data stored in relational database management systems (RDBMS).

Overview

SQL provides a way to interact with databases through a set of commands and statements. These commands can be classified into several categories, including Data Definition Language (DDL), Data Manipulation Language (DML), Data Control Language (DCL), and Transaction Control Language (TCL).

Data Definition Language (DDL)

DDL statements are used to define the database structure. Common DDL commands include:

  • CREATE: Used to create databases, tables, indexes, and other database objects. 
    CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    Name VARCHAR(100),
    Age INT,
    GPA DECIMAL(3, 2)
);
  • ALTER: Used to modify existing database objects. 
    ALTER TABLE Students
ADD COLUMN Major VARCHAR(50);
  • DROP: Used to delete database objects. 
    DROP TABLE Students;

Data Manipulation Language (DML)

DML statements are used to manipulate data within the database. Common DML commands include:

  • SELECT: Used to retrieve data from one or more tables. 
    SELECT Name, Age, GPA
FROM Students
WHERE GPA > 3.5;
  • INSERT: Used to add new rows to a table. 
    INSERT INTO Students (StudentID, Name, Age, GPA)
VALUES (1, 'John Doe', 20, 3.7);
  • UPDATE: Used to modify existing rows in a table. 
    UPDATE Students
SET GPA = 3.8
WHERE StudentID = 1;
  • DELETE: Used to remove rows from a table. 
    DELETE FROM Students
WHERE StudentID = 1;

Data Control Language (DCL)

DCL statements are used to control access to data in the database. Common DCL commands include:

  • GRANT: Used to give users access privileges to the database. 
    GRANT SELECT, INSERT ON Students TO 'username';
  • REVOKE: Used to remove access privileges from users. 
    REVOKE INSERT ON Students FROM 'username';

Transaction Control Language (TCL)

TCL statements are used to manage transactions in the database. Common TCL commands include:

  • COMMIT: Used to save the changes made in the current transaction. 
    COMMIT;
  • ROLLBACK: Used to undo the changes made in the current transaction. 
    ROLLBACK;
  • SAVEPOINT: Used to set a savepoint within a transaction. 
    SAVEPOINT savepoint1;

Joins

Joins are used to combine rows from two or more tables based on a related column between them. Common types of joins include:

  • INNER JOIN: Returns only the rows that have matching values in both tables. 
    SELECT Students.Name, Enrollments.CourseID
FROM Students
INNER JOIN Enrollments ON Students.StudentID = Enrollments.StudentID;
  • LEFT JOIN: Returns all rows from the left table and the matched rows from the right table. 
    SELECT Students.Name, Enrollments.CourseID
FROM Students
LEFT JOIN Enrollments ON Students.StudentID = Enrollments.StudentID;
  • RIGHT JOIN: Returns all rows from the right table and the matched rows from the left table. 
    SELECT Students.Name, Enrollments.CourseID
FROM Students
RIGHT JOIN Enrollments ON Students.StudentID = Enrollments.StudentID;
  • FULL JOIN: Returns all rows when there is a match in either left or right table. 
    SELECT Students.Name, Enrollments.CourseID
FROM Students
FULL OUTER JOIN Enrollments ON Students.StudentID = Enrollments.StudentID;

Applications

  • Database Management: SQL is used to create, maintain, and query relational databases.
  • Data Analysis: SQL is used by data analysts to retrieve and analyze data from databases.
  • Web Development: SQL is used to interact with databases in web applications.
  • Business Intelligence: SQL is used to extract and manipulate data for reporting and decision-making.

Conclusion

SQL is a powerful and versatile language for managing and manipulating relational databases. By understanding and using SQL commands, one can efficiently interact with databases, retrieve meaningful data, and perform various operations essential for data management and analysis.
